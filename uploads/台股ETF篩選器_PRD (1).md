# PRD｜台股 ETF 篩選器網站
**版本**：v1.0  
**日期**：2026-05-11  
**類型**：純前端靜態網站  
**目標用戶**：股票小白、ETF 入門投資者

---

## 一、產品目標

打造一個簡潔、易用的台股 ETF 總覽與篩選工具，讓不熟悉股市的使用者也能快速找到符合需求的 ETF，並理解其基本特性。

---

## 二、技術規格

| 項目 | 規格 |
|---|---|
| 類型 | 純前端靜態網站（HTML / CSS / JavaScript） |
| 資料來源 | 靜態 JSON 檔（手動維護） |
以投信投顧公會為主查每檔 ETF 的基本資料（名稱、發行商、類型）
TWSE + TPEX 補充上市/上櫃分類與官方連結
| 部署方式 | 可放 GitHub Pages / Netlify / 任何靜態主機 |
| 語言 | 繁體中文 |
| 響應式 | 支援桌機、平板、手機 |
| 外部依賴 | 無後端、無資料庫；可使用 CDN 引入前端框架（如 Vue.js / Alpine.js）與 CSS 框架 |

## 三、資料結構

每筆 ETF 資料包含以下欄位，儲存於 `data/etf.json`：

```json
{
  "id": "0050",
  "name": "元大台灣50",
  "code": "0050",
  "asset_class": "股票",
  "asset_mid": "區域",
  "asset_subclass": "股票-區域-新興亞太-台灣",
  "investment_type": "市值型",
  "active_passive": "被動",
  "leverage": "一般",
  "theme_factor": [],
  "dividend_months": [1, 4, 7, 10],
  "top5_holdings": ["台積電", "鴻海", "聯發科", "廣達", "中信金"],
  "investment_direction": "追蹤台灣50指數，涵蓋台灣市值前50大上市公司，集中於半導體與科技產業。",
  "official_url": "https://www.yuantafunds.com/etf/0050"
}
```

### 欄位說明

| 欄位 | 型別 | 說明 |
|---|---|---|
| `id` | string | 唯一識別碼（同代碼） |
| `name` | string | ETF 中文全名 |
| `code` | string | 股票代碼（如 0050） |
| `asset_class` | string | 資產大類（股票/債券/原物料/外匯/槓桿反向） |
| `asset_mid` | string | 資產中類（地區/產業/風格/債券種類/區域/存續期間/貴金屬等） |
| `asset_subclass` | string | 資產細類（如 股票-地區-台灣） |
| `investment_type` | string | 投資分類（見下方列舉） |
| `active_passive` | string | `主動` / `被動` |
| `leverage` | string | `一般` / `2x槓桿` / `-1x反向` / `-2x反向` |
| `theme_factor` | string[] | 主題或因子標籤（可多個） |
| `dividend_months` | number[] | 配息月份，空陣列代表無配息 |
| `top5_holdings` | string[] | 前五大持股名稱 |
| `investment_direction` | string | 投資方向文字說明（100字以內） |
| `official_url` | string | 官方資訊連結 |

## 五、功能規格

### 5.1 Header

- 網站名稱：**台股 ETF 總覽**（或可討論名稱）
- 副標語：「輕鬆找到適合你的 ETF」
- 顯示目前資料庫中的 ETF 總數（如：共 XXX 支 ETF）

---

### 5.2 篩選器（左側欄）

#### 行為規則
- 各篩選項目之間：**AND**（同時符合所有選取條件）
- 同一篩選項目內多選：**OR**（符合其中一個即可）
- 點選已選取的選項可取消
- 提供「清除所有篩選」按鈕
- 篩選結果即時更新表格（無需按確認）

#### 篩選項目

**1. 關鍵字搜尋**
- 輸入框，支援模糊搜尋
- 比對範圍：ETF 名稱、代碼

**2. 資產類別**（三層結構，Accordion 展開）

採財經M平方三層階層式篩選：大類 → 中類 → 細類，均可多選 OR。

**互動行為**：
- 點大類標題旁箭頭 → 展開中類（Accordion）
- 點中類 → 展開細類 Checkbox 列表

- 點中類 Checkbox → 全選該中類所有細類

- 提供「清除所有篩選」按鈕

### 5.3 ETF 表格（右側主內容）

#### 顯示規則
- 預設顯示全部 ETF，依代碼升冪排列
- 手機版表格可橫向捲動（`overflow-x: auto`）
- 每頁顯示 **25 筆**，底部有分頁器
- 表格列 hover 時有輕微背景色變化

#### 欄位（由左至右）

| # | 欄位名稱 | 寬度建議 | 說明 |
|---|---|---|---|
| 1 | 收藏 | 48px | ☆/⭐ 星星圖示，固定不捲動（sticky） |
| 2 | 代碼 | 80px | 如 0050，固定不捲動（sticky） |
| 3 | ETF 名稱 | 160px | 中文全名 |
| 4 | 資產類別 | 100px | 文字標籤 |
| 5 | 投資分類 | 100px | 文字標籤 |
| 6 | 主被動 | 80px | 文字標籤 |
| 7 | 槓桿/反向 | 100px | 文字標籤 |
| 8 | 主題/因子 | 150px | 多個 Tag 顯示 |
| 9 | 配息月份 | 200px | 12格月份圖示（見下方說明） |
| 10 | 前五大持股 | 200px | 逗號分隔文字 |
| 11 | 投資方向說明 | 250px | 截斷顯示，hover 可看完整（Tooltip） |
| 12 | 官方資訊 | 80px | 「查看」按鈕，開新分頁 |

#### 配息月份圖示設計
- 橫排 12 個格子，標示 1～12 月
- **有配息月份**：格子填色（主題色）
- **無配息月份**：格子空心或灰色
- **完全無配息**：全部灰色 + 「無配息」標示

```
範例（季配 1/4/7/10 月）：

```

### 5.4 最愛收藏功能

#### 行為規則
- 表格每列最左側（代碼欄旁）顯示星星圖示
- ☆ 空心 = 未收藏；⭐ 實心填色 = 已收藏
- 點擊星星即時切換狀態，無需確認
- 收藏清單儲存於 **localStorage**，關閉瀏覽器後仍保留

#### 篩選器整合
- 篩選器最上方新增「⭐ 只顯示我的最愛」Toggle 開關
- 開啟時：表格僅顯示已收藏的 ETF，其他篩選條件仍可疊加使用
- 關閉時：恢復顯示全部（依其他篩選條件）

#### 顯示
- Header 或篩選器區塊顯示目前收藏數量（如：「已收藏 5 支」）

---

### 5.5 篩選結果狀態

- 顯示目前篩選結果數量：「找到 XX 支 ETF」
- 若結果為 0：顯示空狀態提示「找不到符合條件的 ETF，請調整篩選條件」

---

### 5.5 手機版（RWD）行為

| 元素 | 手機版行為 |
|---|---|
| 篩選器 | 收合，點擊「篩選」按鈕展開底部 Drawer 或覆蓋層 |
| 表格 | 橫向捲動；「代碼」欄固定（sticky left） |
| 配息圖示 | 縮小格子尺寸 |
| 分頁器 | 簡化顯示（上一頁 / 第N頁/共N頁 / 下一頁） |

## 七、資料維護說明

由於是靜態網站，資料更新流程如下：

1. 編輯 `data/etf.json` 新增或修改 ETF 資料
2. 重新部署靜態檔案（git push 即可若用 GitHub Pages）
3. 無需重新編譯（純 JSON，網頁載入時即時讀取）
